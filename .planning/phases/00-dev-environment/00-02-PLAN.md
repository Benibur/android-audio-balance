  <files></files>
  <read_first>
    - /home/ben/Dev-local/android-audio-controler/.planning/phases/00-dev-environment/00-01-SUMMARY.md (confirms SDK location and build success)
  </read_first>
  <action>
Install emulator and system image, create AVD, boot it, and deploy the app. Exact steps:

1. Install emulator and system image via sdkmanager:
   ```
   export ANDROID_HOME=/home/ben/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator
   sdkmanager "emulator" "system-images;android-35;google_apis;x86_64"
   ```

2. Create an AVD named "dev_phone":
   ```
   echo "no" | avdmanager create avd \
     --name dev_phone \
     --package "system-images;android-35;google_apis;x86_64" \
     --device "pixel_6"
   ```

3. Boot the emulator in background (headless if no display, or with display):
   ```
   $ANDROID_HOME/emulator/emulator -avd dev_phone -no-snapshot-save &
   ```
   Wait for boot: `adb wait-for-device` then poll `adb shell getprop sys.boot_completed` until it returns "1".

4. Install and launch the app:
   ```
   cd /home/ben/Dev-local/android-audio-controler
   adb install app/build/outputs/apk/debug/app-debug.apk
   adb shell am start -n com.audiobalance.app/.MainActivity
   ```

5. Verify the activity is running:
   ```
   adb shell dumpsys activity activities | grep "com.audiobalance.app"
   ```
   This should show the MainActivity in the activity stack.

NOTE: If KVM is not available (`/dev/kvm` missing), the emulator will be very slow. In that case, check `ls /dev/kvm` and if missing, install kvm: `sudo apt-get install -y qemu-kvm` and add user to kvm group: `sudo adduser $USER kvm`. May require logout/login.

  <name>Task 2: Verify physical device ADB connection and deploy APK</name>
  <files></files>
  <action>
Before pausing for human verification, Claude runs these automated steps:
1. `adb devices` to check for a physical device (non-emulator entry)
2. If a physical device serial is found: `adb -s {device_serial} install app/build/outputs/apk/debug/app-debug.apk`
3. If installed successfully: `adb -s {device_serial} shell am start -n com.audiobalance.app/.MainActivity`
4. If no physical device is detected, inform the user they need to connect one and enable USB debugging.

The checkpoint then pauses for the user to confirm the app is visible on their phone screen.

Prerequisites the user must complete before this task:
1. Enable Developer Options on Android phone: Settings > About phone > tap "Build number" 7 times
2. Enable USB Debugging: Settings > Developer Options > USB Debugging > ON
3. Connect phone via USB cable to this computer
4. Accept the "Allow USB debugging?" prompt on the phone if it appears
  </action>
