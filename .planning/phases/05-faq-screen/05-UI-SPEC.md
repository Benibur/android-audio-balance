## Spacing Scale

Declared values (must be multiples of 4 — expressed in dp for Android):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon-to-text gap, inline padding |
| sm | 8dp | Compact element spacing (e.g. between label and content) |
| md | 16dp | Default horizontal screen padding, card internal padding |
| lg | 24dp | Section padding, top of screen |
| xl | 32dp | Major section breaks |
| 2xl | 48dp | Top padding for page-level layouts |
| 3xl | 64dp | Not used in this phase |

Exceptions:
- FAQ item vertical spacing: 12dp between accordion items (matches device list item spacing from DeviceListScreen)
- Footer section top margin: 24dp above the footer GitHub link block

Source: PermissionScreen.kt (48dp top, 32dp bottom, 16dp horizontal, 24dp spacers), DeviceListScreen.kt (12dp vertical item spacing, 16dp horizontal padding)

## Color

This phase uses Material3 dynamic color exclusively — no hardcoded hex values. Colors reference `MaterialTheme.colorScheme.*` tokens.

| Role | M3 Token | Usage |
|------|----------|-------|
| Dominant (60%) | `colorScheme.background` / `colorScheme.surface` | Screen background, accordion item background |
| Secondary (30%) | `colorScheme.surfaceVariant` | Expanded accordion content background |
| Accent (10%) | `colorScheme.primary` | GitHub link text, info icon in DeviceListScreen TopAppBar |
| Destructive | not used in this phase | — |

Accent reserved for:
- The info icon (ⓘ) in DeviceListScreen TopAppBar (tint = `colorScheme.primary`)
- The clickable GitHub URL text in both the open source FAQ answer and the footer section

Secondary color (`onSurfaceVariant`) is used for subdued text: closed accordion question hint, footer icon tint.

Source: DeviceListScreen.kt and DeviceCard.kt established `colorScheme.primary` for active/accent elements, `colorScheme.onSurfaceVariant` for secondary text.

## Screen Layout Contract

### FaqScreen

```
Scaffold
  TopAppBar
    navigationIcon: IconButton(Icons.AutoMirrored.Outlined.ArrowBack)  — navigates back
    title: Text("FAQ")  — titleLarge
  content: LazyColumn
    contentPadding: horizontal=16dp, vertical=12dp
    verticalArrangement: spacedBy(12dp)
    items: 6 × FaqItem (accordion)
    item: FooterSection (GitHub link block)
```

### FaqItem (accordion)

- Collapsed: Row with question text (titleMedium) + trailing expand/collapse icon (Icons.Outlined.ExpandMore / ExpandLess)
- Expanded: Column — question row + AnimatedVisibility content block (bodyLarge answer text, padding 16dp horizontal / 8dp vertical)
- Tap anywhere on the Row (full width) toggles expand/collapse
- Only one item open at a time is NOT required — multiple can be open simultaneously (simpler implementation, no shared state needed)
- Background when expanded: `colorScheme.surfaceVariant`, shape: MaterialTheme.shapes.medium
- Background when collapsed: transparent (inherits surface)
- Expand/collapse uses `AnimatedVisibility` with default fade + expand vertical transition

### FooterSection

- Separator: `Divider()` with top margin 8dp
- Row: GitHub icon (Icons.Outlined.OpenInBrowser, 20dp, tint=primary) + 8dp gap + clickable Text (labelMedium, color=primary, underline decoration)
- URL opens via `Intent(Intent.ACTION_VIEW, Uri.parse("https://github.com/Benibur/android-audio-balance"))`
- Top padding: 24dp before divider

## Accessibility Contract

- All interactive elements have `contentDescription` set
- Info icon in DeviceListScreen: `contentDescription = "About this app"`
- Back button: `contentDescription = "Back"`
- GitHub link: `contentDescription = "Open GitHub repository"`
- Accordion toggle icon: `contentDescription = "Expand"` / `"Collapse"` per state
- Touch targets: minimum 48dp height for each FaqItem row (Material3 minimum touch target)
- Color is never the sole indicator of meaning (expand state also shown by icon change)
