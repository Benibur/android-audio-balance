## Spacing Scale

Declared values in `dp` (Android density-independent pixels, equivalent to CSS px at 1x). All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, chip padding, divider insets |
| sm | 8dp | Between elements within a card (already used in MainActivity `spacedBy(8.dp)`) |
| sm-plus | 12dp | Device list vertical padding (`contentPadding vertical`) and inter-card gap (`Arrangement.spacedBy(12dp)`) — tighter than `md` to keep more cards visible on screen |
| md | 16dp | Card internal padding, screen horizontal padding (already used in MainActivity `padding(16.dp)`) |
| lg | 24dp | Between sections or grouped card clusters if needed |
| xl | 32dp | Permission screen vertical centering padding |
| 2xl | 48dp | Permission screen illustration top margin |
| 3xl | 64dp | Not used in this phase |

Exceptions:
- Touch targets (slider thumb, toggle) minimum 48dp height to meet Android accessibility guidelines (WCAG 2.5.5)
- Slider track full-width inside card horizontal padding: `fillMaxWidth()` with `padding(horizontal = 16dp)` on parent card

## Screen Inventory

### Screen 1: Permission Onboarding (`PermissionScreen`)

**Focal point:** The "Grant permissions" filled button is the single focal element — it sits at the bottom of a centered column against a clear background with no competing interactive elements.

Shown on first launch only. One-shot — never re-shown once all permissions granted.

**Layout:** Single full-screen Composable, centered column.
- Top 48dp margin
- Icon: `Icons.Outlined.Bluetooth` at 64dp, color `colorScheme.primary`
- Title: "Allow Audio Balance to work" — `titleLarge`
- Body: "To detect your Bluetooth headphones and run in the background, this app needs two permissions." — `bodyLarge`
- Permission list (2 items, each with icon + label):
  - `Icons.Outlined.BluetoothConnected` + "Bluetooth Connect — to detect your headphones"
  - `Icons.Outlined.Notifications` + "Notifications — to show balance status while running"
- CTA button: "Grant permissions" — `Button` (filled), full width
- Bottom margin: 32dp

**Permission request sequence:**
1. Request `BLUETOOTH_CONNECT` — system dialog
2. If granted, immediately request `POST_NOTIFICATIONS`
3. Both granted → navigate to device list (replace backstack)
4. Either denied → show denial state (see below)

**Denial state (inline, same screen):**
- Replace body text with: "Permission denied. Tap 'Open settings' to grant it manually."
- Replace CTA with two buttons stacked: "Open settings" (filled) + "Retry permissions" (outlined)
- "Open settings" launches `Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS)`

### Screen 2: Device List (`DeviceListScreen`)

**Focal point:** The connected device card at index 0 is the dominant element — it renders at full opacity with a `colorScheme.primary` border while all other cards recede at 0.72 alpha, directing attention to the currently active device.

Main screen. Only screen visible after permissions granted.

**Layout:** `Scaffold` with `TopAppBar` + `LazyColumn`.
- `TopAppBar`: title "Audio Balance", no navigation icon, no actions in Phase 3
- `LazyColumn`: vertical list of `DeviceCard` composables, `contentPadding = PaddingValues(horizontal = 16dp, vertical = 12dp)`, `verticalArrangement = Arrangement.spacedBy(12dp)`

**Sort order:** Connected device at index 0. All others sorted by most-recently-connected (DataStore order). Connected device stays pinned at top even during scroll.

**Empty state** (no devices ever connected):
- Centered column in remaining screen space
- Icon: `Icons.Outlined.BluetoothSearching` at 48dp, `colorScheme.onSurfaceVariant`
- Text: "No devices yet" — `titleMedium`
- Subtext: "Connect a Bluetooth device to see it here." — `bodyMedium`, `colorScheme.onSurfaceVariant`

### Component: DeviceCard

`ElevatedCard` with:
- `modifier = Modifier.fillMaxWidth()`
- Border: 2dp `colorScheme.primary` stroke if connected, no border otherwise
- Alpha: `1.0f` if connected, `0.72f` if disconnected
- Internal padding: 16dp all sides

**Card layout (top to bottom):**

Row 1 — Device header:
- Left: device name (`bodyLarge`, weight Normal, single line, ellipsis overflow)
- Right (if connected): `SuggestionChip` with label "Connected", `Icons.Filled.BluetoothConnected` icon, `colorScheme.tertiaryContainer` background
- Right: `Switch` (toggle auto-apply), `checked = device.autoApplyEnabled`

Row 2 — Balance value label:
- Centered text above slider: balance label string (see Copywriting → Balance labels)
- Style: `labelMedium`, `colorScheme.primary` if non-zero, `colorScheme.onSurfaceVariant` if Center

Row 3 — Slider row:
- "L" label (`labelMedium`, `colorScheme.onSurfaceVariant`) — left of slider
- `Slider` (`fillMaxWidth`, weight 1f) — balance slider
- "R" label (`labelMedium`, `colorScheme.onSurfaceVariant`) — right of slider

Slider parameters:
- `value`: Float mapped from [-100, +100] to [0f, 1f] for Compose Slider API, converted back in ViewModel
- `valueRange = 0f..1f`
- `steps = 0` (continuous)
- Magnetic snap: when released and value within ±3 units of 0 (mapped: 0.47..0.53), snap to exact center (0.5f). Implemented via `onValueChangeFinished` callback in ViewModel.
- Real-time: `onValueChange` throttled — send to service at most every 50ms (use `debounce(50)` or timestamp check in ViewModel)
- `onValueChangeFinished`: persist to DataStore via BalanceRepository, apply final value to service without throttle

## Interaction Contracts

### Slider real-time feedback

- **During drag:** ViewModel receives `onValueChange` events, throttles to 50ms, sends `Intent("seed_balance")` with `balance` Float extra to `AudioBalanceService` via `context.startService()`. No binding required for real-time.
- **On release:** `onValueChangeFinished` fires, ViewModel calls `BalanceRepository.saveBalance(mac, value)` + sends one final unthrottled `Intent("seed_balance")`.
- **Magnetic snap:** In `onValueChangeFinished`, if `abs(value) <= 3`, snap to 0. Update slider state and trigger one `seed_balance` Intent with value 0.
- **Snap animation:** `animateFloatAsState(targetValue, animationSpec = spring(dampingRatio = Spring.DampingRatioMediumBouncy, stiffness = Spring.StiffnessLow))` applied to the slider's displayed value during snap only.

### Toggle auto-apply

- `onCheckedChange`: ViewModel saves `autoApplyEnabled` flag to DataStore via `BalanceRepository` (new field). No service call needed — service reads flag at connect time.
- No confirmation dialog. Toggle is immediately effective and immediately persisted.
- Disabled state: when toggle is off, slider is still interactive (user can pre-configure balance), but visual opacity of slider row reduces to `0.5f`.

### Service communication

- **Pattern:** Intent-based (consistent with Phase 2 `seed_balance` handler). No bound service required in Phase 3.
- **UI → Service:** `context.startService(Intent(context, AudioBalanceService::class.java).apply { action = "seed_balance"; putExtra("balance", value) })`
- **Service → UI:** `AudioBalanceService` exposes a `companion object` `StateFlow<ServiceState>` (singleton) that the ViewModel collects via `viewModelScope`. `ServiceState` contains: `connectedDeviceMac: String?`, `connectedDeviceName: String?`, `currentBalance: Float`.
- **ViewModel:** `DeviceListViewModel` collects `AudioBalanceService.stateFlow` and `BalanceRepository.allDevicesFlow()` to produce a merged `StateFlow<DeviceListUiState>`.

### Navigation

- Single Activity (`MainActivity`)
- Navigation Compose (`NavHost`) with 2 destinations: `"permissions"` and `"device_list"`
- Start destination determined at launch: if all required permissions already granted → `"device_list"`, else → `"permissions"`
- Permission screen pops itself off the backstack on success (no back-navigation to permissions once granted)
